# Changelog

## v1.5.3 — 2026-06-02
- 批量新增支援貼上 Excel 資料自動填入欄位
- 超過 10 筆資料自動新增列，無上限限制

## v1.5.2 — 2026-06-02
- 新增批量新增設備功能，最多一次新增 10 台，只需填設備名稱和 IP
- 每列 IP 欄位可即時 Ping 測試，空白列自動略過
- 新增後背景自動執行初始狀態檢查

## v1.5.1 — 2026-06-02
- 首頁與公司頁面設備狀態改為每 10 秒自動更新
- stat 卡片（總數/正常/警告/離線）即時同步
- 斷線設備自動出現在警示列表，恢復後自動消失
- 各公司彙總數字即時更新

## v1.5.0 — 2026-06-02
- 新增設備頁 IP 欄位旁加入 Ping / Traceroute 先期檢測按鈕
- 設備類型預設改為「一般」，排列至第一位
- install.sh 自動安裝 traceroute 系統套件
- update.sh 更新時自動補裝缺少的系統套件

## v1.4.3 — 2026-06-02
- 服務啟動時自動開啟持續監控，不再需要手動開啟

## v1.4.2 — 2026-06-02
- admin 新增重啟服務按鈕（版本號旁），點擊確認後自動重啟並等待服務回來刷新頁面

## v1.4.1 — 2026-06-02
- update.sh 加入自動備份資料庫與失敗自動 rollback 機制
- 備份保留最近 5 份，更新失敗時自動還原程式碼與資料庫並重啟服務

## v1.4.0 — 2026-06-02
- 新增版本更新歷程功能，點擊版本號可查看 changelog
- 版本號旁有新版本時顯示 badge，admin 可一鍵更新系統
- 版本偵測改為每 30 分鐘背景自動輪詢

## v1.3.9 — 2026-06-02
- 新增網頁版本更新提示，有新版本時 navbar 顯示 badge
- admin 可從網頁一鍵觸發系統更新，自動等待服務重啟後刷新
- 版本偵測改為每 30 分鐘背景自動輪詢，不需換頁

## v1.3.8 — 2026-06-01
- 一鍵安裝指令改為 `curl -fsSL ... | bash` 格式
- 斷線紀錄加入斷線原因欄位，顯示是哪種 check 觸發（Ping / SSH / Port）

## v1.3.6 — 2026-06-01
- 設備編輯 modal 加入 SNMP Label 與 OID 欄位
- SNMP 監控顯示 interface 名稱（如 GigEth0/0）

## v1.3.1 — 2026-05-30

## v1.3.0 — 2026-05-28

## v1.1.1 — 2026-05-20
